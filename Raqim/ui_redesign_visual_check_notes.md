# ملاحظات التحقق البصري لإعادة تصميم رقيم

تم فتح الواجهة عبر الرابط المؤقت العام بعد تنفيذ إعادة التصميم. الصفحة الرئيسية تعرض هوية رقيم المبسطة مع اتجاه RTL، وشاشة رفع ملف حديثة، وروابط الدخول/إنشاء الحساب ضمن محتوى الصفحة. شريط التنقل العلوي يظهر اسم رقيم فقط بصورة مبسطة.

تم فتح صفحة تسجيل الدخول بنجاح. الصفحة تظهر بطاقة دخول حديثة باللغة العربية، مع حقول البريد الإلكتروني وكلمة المرور، ورسالة توضّح أن تسجيل الدخول تجريبي مؤقت. لا تظهر أي معلومات تقنية للمستخدم مثل OCR Engine أو Model Name أو Runtime Information.

تم سابقًا فحص ملفات الواجهة نصيًا وتأكيد عدم وجود أسماء ALLaM أو CorpusFilter أو مصطلحات تقنية ضمن نصوص واجهة React.

نتيجة البناء: نجح بناء React باستخدام DISABLE_ESLINT_PLUGIN=true npm run build.


## ملاحظة اختبار الرفع

تمت محاولة رفع صورة اختبار من الواجهة الرئيسية للتحقق من تدفق الوصول إلى المراجعة. عنصر منطقة الرفع الظاهر للمستخدم يعمل كـ label بصري، بينما لم يظهر حقل الملف المخفي ضمن عناصر الرفع التلقائي في أداة الاختبار؛ لذلك لم تُستكمل تجربة الرفع من المتصفح الآلي في هذه المحاولة. هذا لا يشير إلى خطأ بناء، إذ إن بناء React نجح والصفحة تستجيب، لكنه يعني أن التحقق البصري المباشر لصفحة المراجعة عبر تدفق الرفع يحتاج إما اختيار الملف يدويًا أو كشف عنصر الإدخال للاختبار الآلي فقط.

تم فحص DOM وظهر أن حقل الملف موجود كـ input type=file بمعرف fileInput، لكن أداة الرفع الآلي لم تتمكن من استهدافه لأنه غير مُدرج كعنصر تفاعلي مستقل في قائمة عناصر الصفحة. لم يتم تعديل الكود لهذا السبب لأن النطاق الحالي هو تنفيذ التصميم المعتمد وليس تغيير آلية الرفع أو الباكند.

## التحقق من صفحة اختيار التصحيح والمراجعة

تم فتح صفحة اختيار التصحيح بنجاح، وتعرض الفرق بين التصحيح اليدوي والتصحيح الذكي بأسلوب بطاقات واضح. لا تظهر فيها أسماء مصادر الاقتراحات أو معلومات تقنية.

تم فتح صفحة المراجعة اليدوية بنجاح. التصميم الجديد يعرض الملف الأصلي والنص المستخرج جنبًا إلى جنب، مع شريط تقدم للمراجعة وتظليل واضح للكلمات التي تحتاج انتباهًا داخل النص المستخرج. النصوص المعروضة للمستخدم تستخدم مسميات عامة مثل "اقتراحات التصحيح" و"كلمات تحتاج انتباه" دون ذكر أسماء ALLaM أو CorpusFilter أو أي تفاصيل OCR/Runtime.

## اختبار قائمة اقتراحات التصحيح

تم الضغط على كلمة مظللة داخل النص المستخرج وظهرت قائمة صغيرة بجانب الكلمة نفسها، وليست نافذة منبثقة عامة. القائمة تعرض "الكلمة الحالية" و"اقتراحات التصحيح" فقط دون ذكر أسماء مصادر داخلية مثل ALLaM أو CorpusFilter. كما تحتوي على حقل "تصحيح يدوي" وزري "اعتماد التصحيح" و"الكلمة صحيحة". في حالة عدم توفر اقتراح مؤكد، تظهر رسالة مستخدمية عامة تطلب التعديل اليدوي أو اعتماد الكلمة كما هي.
